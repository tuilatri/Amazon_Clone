================================================================================

A. MISSING DUE TO DATABASE SCHEMA LIMITATIONS

   1. New Customers Today
      Issue: SiteUser table lacks created_at field
      Impact: Cannot track customer growth rate
      Fix Required: Database migration + registration flow update
      Priority: LOW (not critical for daily operations)

   2. Low Stock Alerts
      Issue: ProductItem only has boolean is_in_stock, no quantity
      Impact: Cannot warn about low inventory
      Fix Required: Add stock_quantity field, update inventory logic
      Priority: MEDIUM (useful for e-commerce, but not admin overview scope)

B. NICE-TO-HAVE MISSING FEATURES

      Status: Backend can support, not implemented in frontend
      Priority: LOW (revenue breakdown partially addresses this)

   2. Stale Pending Orders Alert
      What: Highlight if Pending orders > 3 days old
      Status: Fully feasible with current data
      Priority: LOW (admins can see dates in the table)

   3. Charts/Graphs
      What: Visual trends over time
      Status: Would require chart library (recharts/chart.js)
      Priority: LOW (adds complexity, marginal benefit)

C. SCALABILITY CONSIDERATIONS

   Current Implementation: Works well for small-to-medium order volumes
   
   Potential Issues at Scale:
   - Status counts query on every refresh (GROUP BY on large table)
   - Revenue calculations iterate over all delivered orders
   
   Mitigation: Pre-aggregate stats in scheduled job (not needed now)
